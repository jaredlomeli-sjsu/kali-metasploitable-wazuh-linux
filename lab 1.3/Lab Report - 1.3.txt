Lab 1.3: Password Attack with Hydra
CompTIA SEC+ Hands-On Lab, Domain 1: Threats, Attacks & Vulnerabilities
Prepared by: Jared Lomeli
Date: June 19, 2026
1. Objective
The purpose of this lab was to perform an automated password attack against an FTP service using Hydra, a tool that rapidly tests large numbers of username and password combinations. SEC+ covers brute force and dictionary attacks extensively under Domain 1, and this exercise demonstrates in concrete terms why weak or predictable credentials remain one of the most common ways attackers gain initial access to a system.
2. Lab Environment
This lab ran inside the isolated Host-Only network (192.168.56.0/24). Kali Linux served as the attacking workstation, targeting Metasploitable 2's FTP service.
Kali Linux: the attacking workstation, used to run Hydra.
Metasploitable 2: the target at 192.168.56.101, which exposes an FTP service on port 21 protected only by weak default credentials.
3. Step-by-Step Procedure
I confirmed that Metasploitable 2 was running and reachable, and noted that it exposes an FTP service on port 21.
I opened a terminal in Kali and prepared to brute-force the FTP login using a known username, msfadmin, paired against a large password wordlist.
I ran Hydra against the target, pointing it at the FTP service and supplying the rockyou wordlist as the password source.
I watched Hydra cycle through password attempts against the msfadmin account until it found a valid combination.
Hydra reported a successful login once it matched the correct password, confirming the account was protected only by a weak, easily guessed credential.
4. Commands & Actions Performed
hydra -l msfadmin -P /usr/share/wordlists/rockyou.txt.gz 192.168.56.101 ftp
This runs Hydra against the FTP service on 192.168.56.101 using a fixed username (msfadmin, via -l) and a password list (rockyou.txt.gz, via -P). Hydra tries each password in the list against the service until one succeeds or the list is exhausted.
5. Network / Flow Diagram

6. Results & Observations
Hydra successfully cracked the msfadmin account, reporting the result as [21][ftp] host: 192.168.56.101 login: msfadmin password: msfadmin.
The correct password was found quickly because it was short, common, and identical to the username, which made it an early hit in the wordlist.
No account lockout or rate limiting interrupted the attack, allowing Hydra to make repeated attempts without restriction.
7. What I Learned
This lab made the danger of weak passwords very tangible. Watching Hydra succeed within moments against a simple, predictable password reinforced why SEC+ puts so much weight on password complexity requirements and account lockout policies under the Implementation domain. It also highlighted that brute force and dictionary attacks are not purely theoretical, they are fast, automated, and require almost no skill to execute once a tool like Hydra is pointed at an exposed service.
The practical defense is layered: strong, unique passwords, lockout thresholds after a small number of failed attempts, and ideally multifactor authentication, so that a guessed password alone is not enough to gain access.
